Module 2 Lesson One: Intro to Games and Animations

Learning Goals

This module starts with a basic app called CoordXplore and then asks you to modify the app several times. By going through these exercises you will develop a good understanding of the following:

  • Timers, Intervals, and timer-driven events
  • button click events
  • sensor-driven events
  • properties of sprites, making sprites move
  • canvas coordinates, canvas properties
  • Basics of designing games and animations

Animation

Animation means the transformation of an object over time. Making an object move, rotate, change color are all animations. There are many ways to make sprites move, but in this lesson we'll discuss the ways that are most applicable to game development.

  • Clicking a button, touching or dragging the screen, and tilting the phone are user-initiated events.
  • Sprites colliding with each other or with canvas edges are automatic events.
  • Clock components and sprite components have a TimerInterval property. Every time the interval is up, or the timer “ticks”, that triggers a timer event for that object.

    Sprites

    In the pong tutorial, we used two different sprites: the Ball component and the Image sprite (the paddle was this type of sprite). An image sprite is similar to a ball in that it can be moved around on a canvas, but it is visually represented by a custom Image that you upload. These are the only App Inventor components that have the ability to move around on a canvas. In the previous section we explored the canvas coordinate system.
