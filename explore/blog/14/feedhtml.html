You can try a preview of the new system now by visiting <a href="http://ai2.appinventor.mit.edu/">http://ai2.appinventor.mit.edu</a>. We are calling this preview <em>App Inventor 2 Alpha</em>, but after the official release, we’ll simply be calling it <em>App Inventor</em>, and the current system will be then called <em>App Inventor Classic</em>.</p> <p>The biggest difference between <em>App Inventor Classic</em> and <em>App Inventor 2</em> is that <em>App Inventor 2</em> runs completely in the browser: you will not need Java in order to run App Inventor and there is no blocks editor to download. Together with that, there are many minor changes in how the blocks look and, in some cases, what they are called. Overall, programming in <em>App Inventor 2</em> will be very much like programming in <em>App Inventor Classic</em>: you can see this for yourself by using the preview version. </p> <p>Projects created with <em>App Inventor Classic</em> cannot run in <em>App Inventor 2</em>, and vice versa. We are not planning to automatically convert all <em>App Inventor Classic</em> projects to <em>App Inventor 2</em>, although we are exploring some tools that will let you convert your projects on your own.</p> <p>Prior to the release of <em>App Inventor 2</em>, we’ll also be making changes in the appinventor.mit.edu website and the App Inventor Forums to make it easier to find resources and to report bugs.</p> <p>For people now using <em>App Inventor Classic</em> (and especially people with plans to teach using it) we will keep the <em>App Inventor Classic</em> system running, just as it is, at least through Summer 2014, although we will not be adding new features or continuing development on it, except for basic upkeep.</p> <p>For people now using <em>App Inventor 2</em>, thank you for helping us test! Keep in mind that the system is still experimental and under development. In particular, expect to see incompatible changes over the coming weeks -- changes that will break your current <em>App Inventor 2</em> projects or even cause them not to load. We’re sorry for that inconvenience, but with our limited development resources, we cannot yet take on the task of ensuring backwards compatibility while meeting our development schedule. In the meantime, if you have projects you want to keep current as <em>App Inventor 2</em> evolves, we suggest that you save screenshots or instructions for yourself to use in reconstructing your project. Once <em>App Inventor 2</em> is released, we’ll implement an automatic project upgrade mechanism for further changes, just as we have with <em>App Inventor Classic</em>.</p> <p>For those people maintaining your own App Inventor releases, remember that the sources are available via <a href="http://appinventor.mit.edu/appinventor-sources/">http://appinventor.mit.edu/appinventor-sources/</a> on GitHub -- the current master branch is for App Inventor 2: We encourage you to experiment with them.</p> <p>We are excited about releasing <em>App Inventor 2</em>, and working hard to meet our end of year deadline.</p> <p>Many thanks for your continued support and enthusiasm!<br /> Andrew McKinney, Lead Developer<br /> &amp; The MIT App Inventor Team</p> </div></div></div> Tue, 15 Oct 2013 19:32:27 +0000 andrew 402 at http://explore.appinventor.mit.edu http://explore.appinventor.mit.edu/blogs/andrew/2013/10/new-app-inventor-coming#comments App Inventor Goes Wireless! http://explore.appinventor.mit.edu/blogs/andrew/2012/12/app-inventor-goes-wireless <div class="field field-name-body field-type-text-with-summary field-label-hidden view-mode-rss"><div class="field-items"><div class="field-item even" property="content:encoded"><p>We are very excited to be able to announce that...</p> <p>Starting with today’s release of MIT App Inventor, v132, you no longer need a USB cable to connect your phone to your computer. We have added the option to connect the phone to the computer wirelessly. Not only does this avoid the cable, it also eliminates the often tedious process (on Windows) of finding, downloading, and installing drivers specific to your device. Now all you need now is a wireless (WiFi) network that your phone or device and your computer can both access.</p> <p>The first step to use App Inventor wirelessly is to download the <a href="https://play.google.com/store/apps/details?id=edu.mit.appinventor.aicompanion2">MIT AICompanion</a> from the Google Play Store. You start this App like any other Android App. The Blocks Editor now has a new “WiFI” choice in the “Connect to Device” menu. When you select that you will receive a six letter code which you enter into the MIT AICompanion to connect the phone to the PC. You are also shown a QR Code which you can use If your phone has a camera and a Bar Code Scanning Application. You can find more information and detailed documentation in the <a href="http://explore.appinventor.mit.edu/content/setup-device-wifi">wireless setup instructions</a>.</p> <p>The USB option is still there, same as before. But we expect that most App Inventor users will use the wireless method, which in general will be easier to set up and will give better performance. There may be some situations where wireless won’t work well -- such as when there’s no local wireless network, or the wireless network has poor performance. But we recommend that everyone start out by trying wireless to see if it works for you.</p> <p>Again, we are enthusiastic about this new feature, and we hope this makes your App Inventor experience easier and more productive.
