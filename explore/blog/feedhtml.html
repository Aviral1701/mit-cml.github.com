Explore MIT App Inventor blogs http://explore.appinventor.mit.edu/blog en New Version of the MIT App Inventor 2 Setup Tools for Windows http://explore.appinventor.mit.edu/blogs/andrew/2015/02/new <div class="field field-name-body field-type-text-with-summary field-label-hidden view-mode-rss"><div class="field-items"><div class="field-item even" property="content:encoded"><p>The MIT App Inventor project has just released version 2.3.0 of the setup tools for App Inventor 2.</p> <p><strong>Who should use these new tools?</strong></p> <p>This new version adds additional flexibility, and fixes several issues that users have reported, and is now the standard MIT App Inventor 2 Windows Tools Installer. Starting now, users who follow the getting started instructions for Windows will obtain version 2.3.0.</p> <p>The new tools make few user-visible changes. If App Inventor is currently working for you with Windows, you need not install the new version. (Leave well enough alone.)</p> <p><strong>Changes for version 2.3.0:</strong></p> <ol><li>Fixed an issue caused when users selected a custom directory for installing the setup tools,, rather than the default. Subsequently running the uninstaller would delete additional files. This should no longer happen with version 2.3.0.</li> <li>Fixed an issue that ins some cases would prevent the aiStarter program from launching the emulator or connecting to a device using the USB cable.</li> <li>Added the ability to select whether to install tools as a single individual user, or for all users (the default). The installer will check and make sure you have the correct privileges to install accordingly.</li> <li>Users can now install the setup tools using, Windows silent install mode. This makes it more convenient for administrators to automate installations for classrooms or labs with many computers.</li> <li>Added improved error checking and user messages.</li> </ol><p><strong>Important:</strong> Users who installed the previous version of the Windows setup tools v2.2 and earlier to a custom install directory <strong>must uninstall those tools manually</strong>, by going to the custom location and deleting the “AppInventor” directory. At this point you can go ahead and run the new installer program following the directions specified.</p> <p>Users should always make sure they have a backup of their data before installing any new software.</p> <p>In general version 2.3 fixes bugs, allows for more control over installation and improves the user experience. As always please let us know if you find any issues or have any questions installing these new tools. By posting a message on the <a href="https://groups.google.com/forum/#!forum/mitappinventortest">MIT App Inventor Forum.</a></p> <p>Happy Inventing!</p> <p>Andrew McKinney<br /> Director of Software Development, MIT App Inventor Project</p> </div></div></div><section class="field field-name-field-genre field-type-list-text field-label-above view-mode-rss"><h2 class="field-label">Genre:&nbsp;</h2><div class="field-items"><div class="field-item even">Blog</div><div class="field-item odd">News</div><div class="field-item even">Announcement</div></div></section> Tue, 03 Feb 2015 21:11:44 +0000 andrew 572 at http://explore.appinventor.mit.edu Verizon Innovative App Challenge announces 2015 Best In State Winners http://explore.appinventor.mit.edu/blogs/josh/2015/01 <div class="field field-name-body field-type-text-with-summary field-label-hidden view-mode-rss"><div class="field-items"><div class="field-item even" property="content:encoded"><p>Congratulations to our collaborators from the Verizon Foundation and the Technology Student Association, and especially to the just announced <a href="http://appchallenge.tsaweb.org/sites/appchallenge.tsaweb.org/files/Best_In_State_Winners_2015.pdf">Best In State winners of the 2015 Verizon Innovative App Challenge</a>.</p> <p>We're excited to learn about the great ideas coming from these winners, and look forward to sending out our MIT App Inventor Master Trainers in just a short while to work with the Best in Nation winners of this year's competition. My research interests lie at the intersection of data science and education. I am currently creating systems to collect data that will help myself and the team better understand how users program with App Inventor’s blocks-based visual programming language, Blockly!</p> <iframe width="560" height="315" src="//www.youtube.com/embed/ILl8EdWEfkU" frameborder="0" allowfullscreen=""></iframe><p>While Blockly is used by millions of users in dozens of projects worldwide, little is known about how people use the blocks. I have created a logging system known as Blocks-Level Analytics (BLA). It logs every step a user takes in her or his app creation process. The basis of my research is that by seeing the entire process (as opposed to just the finished app) programmers go through while creating their programs, in this case apps, we can better understand the thought process of these people as they program.</p> <p>I am currently looking at how errors occur while people program and how these people then resolve the errors (or, in other words, how they debug). BLA enables us to analyze and understand errors in context.
