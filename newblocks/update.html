Welcome to App Inventor "newblocks" pilot, Update Information

Information for version “nb104”

Numerous changes, including new block colors. Option to show warnings on blocks. You can now create blocks by typing, similar to App Inventor Classic. When connected to the companion app you can change blocks without having the whole app reloaded on the phone. Errors detected on blocks by the companion will be flagged.

IMPORTANT: Projects you export will now end in a ".aia" extension instead of ".zip" (aia for "App Inventor App"). If you import projects, they must end in .aia (note: The internal format remains ZIP).

You will want to update the MIT NewBlocks Companion, though the old version will work (though errors in blocks won't always be flagged).

Information for version “nb103”

Version “nb103” is a minor bugfix update that causes MIT App Inventor version 2 to work properly with Firefox. You do not need to update the MIT New Blocks Companion if you already did so for version “nb102.”

Information for version “nb102”

Welcome to the “nb102” version of App Inventor Newblocks. As of this version we will begin to deprecate the “newblocks” terminology in favor of “MIT App Inventor 2”.

This version requires a new version of the MIT NewBlocks Companion.

This version is a further refinement of the new blocks editing code for MIT App Inventor. Right click on the blocks editor background to see some new features for arranging your blocks. You can also use Chrome to “Zoom” in and out to make building larger apps easier. This version should also work in Firefox (previous versions did not always display the blocks correctly).

This version adds a “Connect to Emulator” button when in the blocks editor. As its name implies, this is a facility that will let you debug your apps using the emulator. Note that this functionality is still being worked on and isn’t finished (or polished that well either!). In particular the UI will likely change.

To use the Emulator you should install the Android SDK or you may be able to use the App Inventor setup tools version of the emulator. In either case you will need to invoke the Android Emulator manually (for now). We recommend using the Android SDK and following the instructions in the SDK documentation. YOU DO NOT NEED TO INSTALL ECLIPSE. The SDK itself is sufficient. You can also start an emulator for any recent version of Android. We recommend using Gingerbread (2.3) or later.

Once you have an emulator running, you need to install the MIT NewBlocks Companion in the emulator. You can use the “adb” command to do this. In particular the command is:

“adb install MITNewBlocksCompanion.apk”

This assumes that you are working in the directory where you downloaded the Companion. You can use an absolute path to the APK file as well.

Once you have done this, you can use the emulator for App Inventor Development. There is one additional catch. Each time you start the emulator you should issue the command:

“adb forward tcp:8001 tcp:8001”

This sets up the network path for the Companion app. Once the emulator is running and you have issued the adb command above, start the Companion App in the emulator. It should recognize that it is running in the emulator and not put up any buttons, etc. You should now (finally!) be able to click the “Connect to Emulator” button in the blocks editor. Your App should now appear in the Companion on the emulator, if all went well.

Obviously we will streamline this process in the future.

Enjoy. Please send any feedback via the Feedback Form at the top of the page (in MIT App Inventor).
